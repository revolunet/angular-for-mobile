AngularJS performance


Angular uses dirty checking to update the views.

1 template binding = 1 check function executed at least twice on each $digest. same for $watch expressions.

$digest triggered by user events, http calls, promises... 
<input type="text" ng-model="company" placeholder="company">
<hr>
Hi, {{ company }}<hr> 
Uppercase : {{ company|uppercase }}<hr>
If user types "Nuxeo", this leads to (5*2) $digests so 10*(NB_BINDINGS+NB_FILTERS) functions calls in this simple example : 30 calls. The whole view bindings/filters are avaluated on each $digest.

Listen $routeChangeStart event to update that direction

Transition your views 2/2 (ng@1.2)

// define routes (note the depth property)
app.config(function($routeProvider) {
  $routeProvider.when('/', {templateUrl: 'home.html', depth:1})
                .when('/products', {templateUrl: 'products.html', depth:2})
                .when('/products/category', {templateUrl: 'category.html', depth:3})
});

// change direction property when route changes
app.run(function($rootScope, $window) {
  $rootScope.direction = 'ltr';
  $rootScope.$on('$routeChangeStart', function(event, next, current) {
    $rootScope.direction = (current && next && (current.depth > next.depth)) ? 'ltr':'rtl';
  });
});
// defines different CSS for each direction
.rtl .view-animate.ng-enter {transform: translate3d(100%,0,0);}
.rtl .view-animate.ng-leave {transform: translate3d(-100%,0,0);}
.ltr .view-animate.ng-enter {transform: translate3d(-100%,0,0);}
.ltr .view-animate.ng-leave {transform: translate3d(100%,0,0);}

Bindonce, self-destruct databinding



Bind your date/promises once, then remove the watchers. 
<ul>
  <li bindonce ng-repeat="person in Persons">
      <a bo-href="'#/people/' + person.id"><img bo-src="person.imageUrl"></a>
      <a bo-href="'#/people/' + person.id" bo-text="person.name"></a>
      <p bo-class="{'cycled':person.generated}" bo-html="person.description"></p>
  </li>
</ul>
result : 1 watcher instead of 1 + (6*n)

https://github.com/Pasvaz/bindonce

angular-gestures, touchy interfaces



Simple wrapper around hammer.js 
<div style="transform:rotate({{rotation}}deg) scale({{scaleFactor}});"
      hm-pinch="pinch($event)" hm-rotate="rotate($event)"></div>
$scope.rotate = function(event) {
  $scope.rotation = event.gesture.rotation % 360;
  event.gesture.preventDefault();
}
$scope.pinch = function(event) {
  $scope.scaleFactor = event.gesture.scale;
  event.gesture.preventDefault();
}

Tips (1/2)



Minimize DOM size, Minimize # of http calls (high latency)

Preload templates (grunt-html2js)

Dont use ngClick but @ftlabs FastClick

Beware of uneeded watchers

use ng-boilerplate and automate all the things with grunt : watch, autoprefixer, cordova build...

Prevent page zooming with 
<meta name=viewport content=width=device-width,user-scalable=no,initial-scale=1,maximum-scale=1>

Tips (2/2)



Use CSS flexboxes for your base layout

Use CSS :active selector

Use CSS3 spinners / RequestAnimationFrame

Use SVG fonts (grunt-webfont)

Use lightweight CSS components : Topcoat.io

Always Use HW-accel CSS for transition

Debug with Safari to have a more real iOS experience

Cordova (Phonegap) integration



Cordova is now a clean and solid platform

Make the app testable in Chrome/Safari : setup a service that dispatch calls depending on the environnement

Use btford.phonegap.ready to queue pending calls before the deviceready event;

Wrap any Cordova callback in a safeApply call to trigger the $digest in both environnements. (conditionnal $apply())

Integrate cordova prepare, build in your grunt workflow

Last but not least



Your server APIs dictates your app complexity

✔ Have clean, well documented APIs (see apiary.io)

✔ Use API best practices (see apigee.com)

✔ Use $httpBackends mocks for parallel development

✔ Don't let backend devs pollute your client app, keep complexity on the server :)
