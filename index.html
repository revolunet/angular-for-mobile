Transition your views 1/2 (ng@1.1.5)

  .when('/contact', { templateUrl: 'contact.html', depth: 1 })
  .when('/contact/help', { templateUrl: 'contact-help.html', depth: 50 });

Transition your views 2/2 (ng@1.5)

var service = {
    getViewAnimation: function() {
        return {
            enter: 'slide-in-' + currentDirection,
            leave: 'slide-out-' + currentDirection
        };
    },
    go: function(route, direction) {
        // guess the animation direction if not provied
        if (!direction) {
            direction = 'rtl';
            var currentRouteDepth = $route.current.depth || Number.MIN_VALUE,
                nextRouteDepth = $route.routes[route] || Number.MAX_VALUE;
            if (currentRouteDepth && nextRouteDepth) {
                direction = (nextRouteDepth >= currentRouteDepth)?'rtl':'ltr';
            }
        }
        currentDirection = direction;
        // finally redirect to the desired view
        $location.path(route);
    }
};

Bindonce, self-destruct databinding



Bind your date/promises once, then remove the watchers. 
<ul>
  <li bindonce ng-repeat="person in Persons">
      <a bo-href="'#/people/' + person.id"><img bo-src="person.imageUrl"></a>
      <a bo-href="'#/people/' + person.id" bo-text="person.name"></a>
      <p bo-class="{'cycled':person.generated}" bo-html="person.description"></p>
  </li>
</ul>
result : 1 watcher instead of 1 + (6*n)

https://github.com/Pasvaz/bindonce

angular-carousel, uses $swipe



How carousel uses the $swipe service 
$swipe.bind(carousel, {
  'start': function(coords) {
    // store coords
  },
  'move': function() {
    // move carousel to follow the finger
  },
  'end': function() {
    // animate to prev/next slide, rebuffer the DOM...
  }
}

Tips (1/2)



Take care of CSS (or use TopCoat.io)

Use HW-accel CSS for transition

Beware of uneeded watchers

Prevent page zooming with 
<meta name=viewport content=width=device-width,user-scalable=no,initial-scale=1,maximum-scale=1>
Don't use checkbox/radios (slows on iOS), prefer custom inputs (release soon)

Tips (2/2)



Use CSS flexboxes for your base layout

Use CSS3 spinners / RequestAnimationFrame

Dont use ngClick but @ftlabs FastClick

You can use hammer.js (see angular-gestures)

Debug with Safari to have a more real iOS experience

use ng-boilerplate and automate all the things with grunt : watch, autoprefixer, cordova build...

Last but not least



Your server APIs dictates your app complexity

✔ Have clean, well documented APIs (see apiary.io)

✔ Use API best practices (see apigee.com)

✔ Use $httpBackends mocks for parallel development

✔ Don't let backend devs pollute your client app, keep complexity on the server :)
